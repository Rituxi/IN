# AGENTS.md

本项目给 AI IDE、Agent、自动化脚本的维护约束如下。

## 编码与文本

- 所有源码、配置、文档统一使用 `UTF-8` 编码。
- 不要把 UTF-8 文件另存为 ANSI、GBK、UTF-16。
- 修改包含中文的文件前，先确认编辑器当前编码是 `UTF-8`。
- 如果发现中文已经乱码，不要在乱码原文上继续局部拼接，优先整段重写为正常中文。
- 不要批量执行会改变整仓库编码的脚本。

## 查找与阅读

- 优先使用 `rg` / `rg --files` 查找内容和文件。
- 如果当前环境不能正常使用 `rg`，再退回 PowerShell 的 `Get-ChildItem` 和 `Select-String`。
- 先读相关文件，再改代码，不要凭猜测直接重写接口或数据结构。

## 编辑方式

- 小改动优先精确修改，避免无关格式化。
- 大文件如果已经出现乱码，可以按模块整段重写，但必须保持原有接口、路由、请求参数和返回结构兼容。
- 不要随意改动 API 路径、字段名、Redis key、环境变量名。
- 修改 UI 时，优先抽出统一的颜色、按钮、表格样式，避免每页各写一套。

## 中文与文案

- 中文文案要求简洁、自然、面向普通用户。
- 不要引入半截乱码、问号乱码、混乱标点。
- 新增中文时，优先使用简体中文，保持术语前后一致。

## 安全边界

- 不要执行会覆盖用户现有改动的危险命令。
- 不要使用 `git reset --hard`、`git checkout --` 之类的破坏性命令，除非用户明确要求。
- 后台相关改动不能破坏现有登录、用户管理、兑换码、配额配置功能。

## 提交前检查

- 至少执行一次类型检查或构建检查。
- 检查主要页面中文是否正常显示。
- 检查按钮、表格、输入框是否还能正常交互。
